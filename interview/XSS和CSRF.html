# XSS和CSRF

# XSS

Cross-Site Scripting 跨站脚本攻击，为了和布局的那个CSS区分才这么简写。

是一种代码注入攻击 。

用户UGC输入、URL参数、post参数、第三方链接、Refer、Cookie都不可信任。

# XSS攻击的方式及应对方法：

  • 特殊的HTML标签符号。escapeHTML转义<>"'/&这些字符。
  • 链接跳转。对于链接跳转，如 <a href="xxx" 或 location.href="xxx"，要检验其内容，禁止以 javascript: 开头的链接，和其他非法的scheme。
  • 内联的JSON。当 JSON 中包含 U+2028 或 U+2029 这两个字符时，不能作为 JavaScript 的字面量使用，否则会抛出语法错误。当JSON中包含字符串 </script> 时，当前的script标签将会被闭合，后面的字符串内容浏览器会按照 HTML 进行解析；通过增加下一个 <script> 标签等方法就可以完成注入。
  • Vue中的v-html指令。底层是使用了innerHtml，使用lint工具禁止使用这个指令，使用插值{{ value }}的方式来渲染。

# CSRF

Cross Site request forgery 跨站请求伪造攻击。 是一种挟制用户在当前已登录的 Web 应用程序上执行非本意的操作的攻击方法

# CSRF攻击的方式

# get型

 <img src="http://bank.example/withdraw?amount=10000&for=hacker" />

在受害者访问含有这个img的页面后，浏览器会自动向http://bank.example/withdraw?account=xiaoming&amount=10000&for=hacker发出一次HTTP请求。bank.example就会收到包含受害者登录信息的一次跨域请求。

# 链接类型

  <a href="http://test.com/csrf/withdraw.php?amount=1000&for=hacker" taget="_blank">
  重磅消息！！
  </a>
