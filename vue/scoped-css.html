# scoped

Vue.js的单文件组件（SFC）和一些基于 Web Components 的框架中， <style> 标签上，scoped属性用于将样式局限在当前的组件或页面中，防止样式影响其他组件或页面。 在使用 scoped 属性后，样式规则只会应用于当前组件或页面中的元素，而不会影响其他地方的相同元素。这种方式避免了全局样式的冲突，使得组件的样式更加模块化、独立和可维护。

scoped属性在标准的 HTML 和 CSS 中并不存在，它是由 Vue.js 和某些 Web Components 框架所支持的特性。如果你在普通的 HTML 页面中使用了 scoped 属性，它将被忽略，不会起到预期的效果。

# 原理

示例：

<template>
  <div>
    <h1>Hello, World!</h1>
    <p>This is a scoped style example.</p>
  </div>
</template>

<style scoped>
h1 {
  color: red;
}

p {
  font-size: 18px;
}
</style>

编译后：

<div data-v-f3f3eg4>
  <h1 data-v-f3f3eg4>Hello, World!</h1>
  <p data-v-f3f3eg4>This is a scoped style example.</p>
</div>

<style>
h1[data-v-f3f3eg4] {
  color: red;
}

p[data-v-f3f3eg4] {
  font-size: 18px;
}
</style>
  • 加了scoped，编译后PostCSS给一个组件中的所有dom添加了一个独一无二的标签属性data-v-hash。
  • 每一个选择器的最后一个选择器单元增加一个属性选择器原选择器[data-v-实例标识]，示例：假设原选择器为.cls #id > div，则更改后的选择器为.cls #id > div[data-v-e0f690c0]；

# 作用范围

  • 组件内部所有DOM，
  • 如果有子组件的话，只作用于子组件根标签
  • 无法作用于子组件内根标签以外的标签

# 穿透

当我们引入第三方组件库时，需要在组件中修改第三方组件的样式，第三方组件使用了scoped属性，删除scoped属性会造成组件之间的样式覆盖，不是明智的选择。这时我们可以通过特殊的方式穿透scoped。

深度选择器：::v-deep(/deep/)、>>>。和scoped一样，这些都是Vue.js中的概念，不是HTML或CSS规范。

<style scoped>
::v-deep h1 {
  color: red; /* 这样会影响子组件中的 h1 元素样式 */
}
</style>

编译结果：

<style>
[data-v-f3f3eg4] h1 {
  color: red; /* 这样会影响子组件中的 h1 元素样式 */
}
</style>

编译过程会给组件的作用域样式 <style scoped> 的每一个深度作用选择器前面的一个选择器单元增加一个属性选择器[data-v-实例标识]，示例：假设原选择器为 .cls #id >>> div，则更改后的选择器为.cls #id[data-v-e0f690c0] div；
