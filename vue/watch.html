# watch（侦听器）原理

# 用法

有三种用法

  1. 直接函数
new Vue({
    data() {
        return { firstName: 'hua' }
    },
    watch: {
        firstName(newValue, oldValue) {
            console.log('1', oldValue, newValue);
        }
    }
})
  1. 数组
new Vue({
    data() {
        return { firstName: 'hua' }
    },
    methods: {
        modify: (newValue, oldValue) => {
            console.log('2-1', oldValue, newValue);
        }
    },
    watch: {
        firstName: [
            'modify',
            (newValue, oldValue) => {
                console.log('2-2', oldValue, newValue);
            },
        ]
    }
})
  1. 实例方法
vm = new Vue({
    data() {
        return { firstName: 'hua' }
    },
});
vm.$watch('firstName', function (newValue, oldValue) {
    console.log('3', oldValue, newValue);
} )

# 前两种用法是第3.中的语法糖。

initState => initWatch(vm.options.watch) => createWatcher => this.$watch

初始化函数initState中，如果定义了watch属性则调用initWatch初始化watch， initWatch中，遍历所有watch对象的key值（和数组每个响应方法），为每个监听响应方法调用createWatcher createWatcher中，返回调用vm.$watch

# 原理

即vue.$watch实例方法的原理。

实例方法，当然是挂在类的prototype上：

Vue.prototype.$watch = function(expOrFn, cb, options = {}) {
  const vm = this
  
  options.user = true  // user-watcher的标志位，传入Watcher类中
  const watcher = new Watcher(vm, expOrFn, cb, options)  // 实例化user-watcher
}

Watcher类中user-watcher（用户的watch）的代码：

class Watcher {
    constructor(vm, exprOrFn, options, cb) {
        this.getter = parsePath(expOrFn); // getter意味着调用这个函数可以发生取值操作
        this.user = options.user; // 标识这是一个user-watcher
        this.cb = cb;
        this.value = this.get();
    }
    get() {
        Dep.target = this;
        pushTarget(this);
        let value = this.getter.call(this.vm); // 会去vm上取值，触发值的响应式getter，收集依赖
        popTarget();
        return value;
    }
    update() { // 属性每被修改一次，使用属性的组件对应的watcher的update都会被调用一次
        if(this.sync) {  // 如果有设置sync为true
            this.run()  // 不走nextTick队列，直接执行
        } else {
            queueWatcher(this)  // 否则加入队列，异步执行run()
        }
    }
    run() {
        const oldValue = this.value;
        const newValue = this.get();
        if (this.user) { // 执行监听对应的回调函数
            this.cb.call(this.vm, newValue, oldValue);
        }
    }
}

解释：

  1. 初始化流程
  • 构造函数中，先搞定user-watcher的getter， 第二个参数名为exprOrFn即表达式和函数，顾名思义包含两种情况。 如果是非函数（a.b.c,firstName），parsePath函数转换，返回一个函数作为getter； 如果是函数，直接就赋给getter。
  • 调用this.get()=>this.getter()，回去进行取值（用户watch中的key值）操作，触发响应式值对应的getter, 依赖收集添加当前user-watcher。返回结果作为oldValue。
  1. 修改侦听值
  • 触发update => run => get => getter，获取被侦听属性的新值；
  • 根据初始化的user标记判断是user-watcher，执行对应的回调函数。
