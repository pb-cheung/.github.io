# Vue异步更新

Vue有响应式的特性，即修改数据后视图会自动刷新，但是这个刷新不是立即就执行的，它是异步的。

视图更新在下一个循环中tick执行，这里利用了事件循环机制。

# watcher队列

如果同个属性在同一事件循环中被多次修改，那么对应的视图（渲染watcher的update方法）会被多次触发更新，这样会造成不必要的计算和DOM操作。如下例：

const vm = new Vue({
    data: () => {
        return {
            message: 'hello vue'
        }
    }
})
vm.message = 'new world'
vm.namessageme = 'haha'

为了避免这种情况，Vue在此处使用了队列queue，每个watcher实例有一个唯一的ID，以此作为映射表的key，来保障同个watcher只能入队一次。

然后在下一个tick调用清空队列函数flushSchedulerQueue，执行watcher的更新（视图更新or回调）。

源码：

let queue = [];
let has = {};
let pending = false; // 防抖

function flushSchedulerQueue() {
    let flushQueue = queue.slice(0);
    queue = [];
    pending = false;
    flushQueue.forEach(w => w.run());
}
// 解决watcher.update被重复调用的问题
// 使用暂存队列，异步更新
function queueWatcher(watcher) {
    const id = watcher.id;
    if (!has[id]) {
        queue.push(watcher); // 暂存，并不执行
        has[id] = true;
        // 无论update执行多少次，但是最终只执行一次刷新操作 （属性短时间内被修改多次）

        if (!pending) {
            // flushSchedulerQueue 放入宏任务队列，同步代码执行完毕后执行
            // setTimeout(flushSchedulerQueue, 0);
            nextTick(flushSchedulerQueue)
            pending = true;
        }
    }
}

# nextTick

如果你想在视图更新后做些事情，比如获取更新后的DOM，就需要用nextTick这个API了(Vue.nextTick(callback)orvm.$nextTick(callback))，参数callback为异步更新后执行的函数。

nextTick主要办了两件事：

  • 采用了优雅降级的方式，选择一个系统支持的API来实现异步（回调函数在下一个tick中执行）
  • 维持一个回调函数队列callbacks，watcher清除调度队列函数flushSchedulerQueue和用户使用nextTick传入参数都可以是callbacks队列中的一个cb

# 异步概念理解测试

<template>
  <div id="app">
    <p ref="name">{{ name }}</p>
    <button @click="handleClick">修改name</button>
  </div>
</template>

<script>
  export default {
  name: 'App',
  data () {
    return {
      name: '井柏然'
    }
  },
  mounted() {
    console.log('mounted', this.$refs.name.innerText)
  },
  methods: {
    handleClick () {
      this.$nextTick(() => console.log('nextTick1', this.$refs.name.innerText))
      this.name = 'jngboran'
      console.log('sync log', this.$refs.name.innerText)
      this.$nextTick(() => console.log('nextTick2', this.$refs.name.innerText))
    }
  }
}
</script>

依次打印：

  1. sync log 井柏然、
  2. nextTick1 井柏然
  3. nextTick2 jngboran

解释：

结合时间循环，对照下图理解

  • 1.比较好理解，同步代码，会先执行
  • 2.和3.放一起理解，示例中的代码在callbacks中添加了三个回调函数[nextTick1Cb, watcher-flushSchedulerQueue, nextTickCb]，下一个tick中按照他们顺序出队执行，nextTick1Cb执行的时候视图还没更新，nextTickCb执行的时候视图已更新所以是修改后的值。
