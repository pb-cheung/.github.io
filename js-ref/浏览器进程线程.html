# 浏览器进程线程

# 进程（Process）和线程

  1. 进程是运行中的应用程序的一个封装。
  2. 进程是资源（CPU、内存等）调度分配的最小单位。（Win和OSX的任务管理器打开，会看到以进程为划分单位的资源占用情况）
  3. 线程是进程的子任务。
  4. 线程是操作系统最小执行和调度单位。
  5. 一个进程可以有多个线程，一个线程只能从属于一个进程。
  6. 进程间通信通过 Inter Process Communication (IPC)机制。

# 浏览器架构

  1. 以Chrome为代表的现代浏览器是多进程的架构（下图右侧）。
  2. 每个浏览器tab对应一个渲染进程（Renderer Process）。
  3. 多进程架构的优点有：其中某个tab未响应，其他tab页则不会受到影响。
