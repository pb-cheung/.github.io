# CommonJS

# 一、特点

  • 来自社区的（非官方的）模块化方案之一，解决的问题是：变量命名冲突、代码复用困难等。
  • 每个文件一个模块，关键API: require导入模块、module.exports和exports导出模块。

# 二、原理

# 变量的注入

CommonJS是一个规范，按照规范书写的模块代码中包含require、module、exports等变量，但是它们并没有被定义，需要运行环境提供。

这是通过包装函数的形式实现的：

function wrapper (script) {
    return '(function (exports, require, module, __filename, __dirname) {' + 
        script +
     '\n})'
}

wrapper函数返回一个函数，module等变量（关键字），以形参的形式传递给一个包装函数，包装函数内容为我们的模块代码。

例如：

// a.js
module.exports = "123"

const modulefunction = wrapper(`
    module.exports = "123"
`)

# require原理

require函数的大致代码：

function require (id) { // id 为模块路径标识符
    const cashedModule = Module.cache[id]
    if (cachedModule) {
        return cachedModule.exports // 如果已有缓存，直接返回缓存的exports
    }

    const module = { exports: {}, loaded: false, ... } // 创建当前模块的module

    Module._cache[id] = module // 模块加入缓存

    // 加载模块
    runInThisContext(wrapper('module.exports = "123"'))(module.exports, require, module, __filename, __dirname, ..) // runInThisContext相当于eval

    module.loaded = true // 加载完成

    return module.exports // 返回值
}

# 三、循环引用问题

CommonJS中使用“延迟加载”的策略来解决循环引用的问题。

模块A依赖模块B，模块B也依赖模块A。 当A通过require引入模块B，模块B还未被加载，会暂时将B标记为未加载状态（module.loaded），并返回一个空对象，然后继续执行A的代码。 当B被其他模块依赖并加载完成后，会将导出的对象（module.exports）缓存起来，并且会更新A中的标记信息(module.loaded)。

举例：

// a.js
const getMes = require('./b')
console.log('this is a.js')
exports.say = function(){
    const message = getMes()
    console.log(message)
}

// b.js
const a = require('./a')
const  object = {
   name:'haha',
   author:'bob'
}
console.log('this is b.js')
console.log('a.js: ' , a)

setTimeout(()=>{
    console.log('async a.js:' , a)
},0)

module.exports = function(){
    return object
}

// main.js
const a = require('./a')
const b = require('./b')

console.log('node 入口文件')

node main.js

# 打印信息
this is b.js
a.js:  {}
this is a.js
node 入口文件
async a.js: { say: [Function (anonymous)] }

# 四、module.exports和exports

exports是module.exports的引用，初始化为对象。

module.exports = exports = {}

# exports的错误用法

// 正确示例
exports.a = 1
exports.say = function() {
    console.log('hello')
}

// 错误示例
exports = {
    a: 1,
    say: function() {
        console.log('hello')
    }
}

因为exports是通过包裹函数传递给模块的变量，错误的实例中，相当于把其重新赋值了，不会再引用原来的形参了。

# module.exports使用场景

需要单独导出一个非对象的情况

module.exports = function() {}

# 五、加载时机

# cjs运行时加载

这里说的动态加载是指在运行时加载。入口代码代码运行，依赖了a模块，此时加载a模块获取其返回值，然后继续运行。

# ESM编译时加载

# import命令提升

import命令具有提升效果，会提升到模块头部先执行。

举例说明：

foo();

import { foo } from 'my_module';

以上示例不会报错，因为import的执行早于foo函数的调用，本质是import是编译阶段执行的。

# import指令不可与表达式一起使用

因为编译阶段表达式无法求值，运行阶段表达式才可以求值

// 报错
import { 'f' + 'oo' } from 'my_module';

// 报错
let module = 'my_module';
import { foo } from module;

// 报错
if (x === 1) {
  import { foo } from 'module1';
} else {
  import { foo } from 'module2';
}

# ESM中动态加载——import()函数

编译阶段固然有利于提高效率，但是无法做到条件加载，无法完全替代CommonJS，ES在后来的提案中新增了import()函数来解决这个问题。

语法：import函数参数为模块地址，返回值为Promise对象

有了它就可以实现按需加载和条件加载

# 按需加载

button.addEventListener('click', event => {
  import('./dialogBox.js')
  .then(dialogBox => {
    dialogBox.open();
  })
  .catch(error => {
    /* Error handling */
  })
});

React中例子：

import { lazy } from 'react';
const MarkdownPreview = lazy(() => import('./MarkdownPreview.js'));

<Suspense fallback={<Loading />}>
  <h2>Preview</h2>
  <MarkdownPreview />
 </Suspense>

来自： https://react.dev/reference/react/lazy#lazy

# 条件加载

if (condition) {
  import('moduleA').then(...);
} else {
  import('moduleB').then(...);
}

# 模块导出区别

  • CommonJS模块导出的是值的拷贝
  • ES Module模块导出的是值的引用

举例1：

// lib.js
var counter = 3;
function incCounter() {
  counter++;
}
module.exports = {
  counter: counter,
  incCounter: incCounter,
};

// main.js
var mod = require('./lib');

console.log(mod.counter);  // 3
mod.incCounter();
console.log(mod.counter); // 3

CommonJS中，lib模块中的counter改变后，main模块中打印看到的值没有变化。

举例2：

// lib.js
export let counter = 3;
export function incCounter() {
  counter++;
}

// main.js
import { counter, incCounter } from './lib';
console.log(counter); // 3
incCounter();
console.log(counter); // 4

ESM中，被依赖模块lib中counter改变后，main模块中打印看到的是改变后的值。
