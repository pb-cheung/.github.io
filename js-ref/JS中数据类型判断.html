# JS中数据类型判断

# typeof

我们可能首先想到的是typeof。但是会出现预期之外的结果。

例如：

console.log(typeof new Date()); // "object"
console.log(typeof new Function()); // "function"
console.log(typeof [1,'a']); // "object"
console.log(typeof Error('test error message')); // "object"
console.log(typeof /[abc]/); // "object"
console.log(typeof null); // "object"

https://jsbin.com/pudarum/1/edit?html,js,console,output

ECMAScript定义了8中数据类型（7种原始类型+Object）。

undefined、null、String、Boolean、Number、Symbol、BigInt ，Object

typeof对7种原始类型(primitive types)的判断没有问题。对其他基于Object实现的对象显示并非预期结果了（如果可调用或者实现了[[call]]属性返回"function"，其他的返回”object“）。

# instanceof

instanceof 运算符检查右值的原型（prototype）是否在左值的原型链上。

let Foo = function() {};
let f = new Foo();

console.log(f instanceof Foo); // true
console.log(f.__proto__ === Foo.prototype); // true
console.log(Object.prototype.getPrototypeOf(o)); // true

console.log(f instanceof Object); // true
console.log(Object.prototype.isPrototypeOf(o)); // true

因此，内置对象的实例，都能判定其继承自哪个对象，例如：

console.log(new Date() instanceof Date); // true
console.log(function (){} instanceof Function); // true

预期之外的情况：

console.log(2 instanceof Number); // false 类型是原始数据类型的Number
console.log(Number(2) instanceof Number); // false 类型转换，类型是原始数据类型的Number
console.log(new Number(2) instanceof Number); // true. 创建了对象实例，类型是Object

# Object.prototype.toString

最佳实践是使用Object.prototype.toString

toString() 返回 "[object type]"，其中 type 是对象的类型。

JavaScript中继承自Object的 诸如String、Array对象等覆盖了这个方法，调用toString()方法返回的基本都是对象的字符串形式，所以不能直接调用toString()来获取类型，要使用Object.prototype.toString.call()的形式。

# 最佳实践代码

jQuery.type()的实现代码：

var class2type = {};
"Boolean Number String Function Array Date RegExp Object Error".split(" ").forEach(function(e,i){
class2type["[object " + e + "]"] = e.toLowerCase();
});

function _typeof(obj){
  if(obj == null){
    return String(obj);
  }
  return typeof obj === "object" || typeof obj === "function" ? class2type[ class2type.toString.call(obj) ] || "object" : typeof obj;
}
