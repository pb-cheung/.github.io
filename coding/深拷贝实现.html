# 手写深拷贝

# 一、浅拷贝

常见的浅拷贝的方法和其对应的问题：

# Object.assign——浅拷贝

较为常用的方法

let obj1 = { a: 'aaa', b: { c: 0 } };
let targetObj = Object.assign({}, obj1);

它的问题：

  1. 无法拷贝Symbol属性
  2. 源对象中的对象属性（obj1.b）无法完成拷贝。拷贝的是对象的引用，修改targetObj.b源对象的对应属性会同时变动。

# JSON.stringify和JSON.parse序列化和反序列化API

能解决大多数场景的问题

问题：

  1. 会忽略undefined、Symbol、函数
  2. NaN、Infinity、-Infinity 会被序列化为 null
  3. 不能解决循环引用问题

循环引用举例：

const obj = {
  a: 1
}

obj.obj = obj

const newObj = JSON.parse(JSON.stringify(obj)) // 报错

# 二、代码实现

function deepClone(target, hash = new WeakMap()) {
    if (target === null) return target
    if (target instanceof Date) return new Date(target)
    if (target instanceof RegExp) return new Range(target)
    if (target instanceof HTMLElement) return target
    
    if (typeof target !== 'object') return target
    
    // 解决循环引用
    if (hash.get(target)) return hash.get(target)
    const cloneTarget = new target.constructor()
    hash.set(target, cloneTarget)
    
    Reflect.ownKeys(target).forEach(key => {
        cloneTarget[key] = deepClone(target[key], hash)
    })
    
    return cloneTarget
}

# 三、解释

  1. null直接返回，接着处理常见的引用类型（日期、正则、DOM对象）;
  2. 其他非引用类型直接返回;
  3. 数组和对象会递归调用深拷贝每一层;
  4. Symbol不会被一般遍历对象的方法（for...in, for...of）遍历到，Reflect.ownKeys()方法可以返回所有类型的键名，包括常规键名和 Symbol 键名;
  5. 循环引用问题，额外开辟一个存储空间来存储当前对象和拷贝对象的对应关系。当需要拷贝当前对象时，先去存储空间中找，有没有拷贝过这个对象，如果有的话直接返回，就不会一直递归导致栈内存溢出了。
  6. 引入WeakMap是为了防止内存泄露。其他位置对target的引用一旦消除，该对象占用的内存就会被垃圾回收机制释放。WeakMap 保存的这个键值对，也会自动消失。
